## 2.0 从零搭建后端项目

### 环境要求

| 工具 | 版本 | 验证命令 |
|------|------|---------|
| Go | >= 1.23 | `go version` |
| MySQL | >= 5.7（InnoDB） | — |
| PostgreSQL（可选） | >= 14 | — |

| GoFrame CLI | 最新版 | `gf version` |
| IDE | GoLand（推荐）或 VS Code + Go 插件 | — |

### 安装 GoFrame CLI

```bash
# 方式一：go install（推荐）
go install github.com/gogf/gf/cmd/gf/v2@latest

# 方式二：从 Release 下载
# https://github.com/gogf/gf/releases

# 验证
gf version
```

### 获取 HotGo 并初始化

```bash
# 1. 克隆 HotGo 仓库
git clone https://github.com/bufanyun/hotgo.git && cd hotgo

# 2. 进入服务端目录
cd server

# 3. 设置国内代理（国内用户）
go env -w GOPROXY=https://goproxy.cn,direct

# 4. 安装 Go 依赖
go mod tidy

# 5. 创建数据库并导入初始数据
# 用 mysql 命令导入 storage/data/hotgo.sql

# 修改 hack/config.yaml 中的 gfcli.gen.dao 数据库连接

# 8. 启动所有服务
gf run main.go
# 或: go run main.go

# 9. 访问 http://localhost:8000/admin
# 默认账号: admin  密码: 123456
```

## 2.2 分层架构

### 核心分层（必须严格遵循）

```
api（HTTP契约） → controller（参数透传） → service（接口） → logic（业务实现） → dao（数据访问）
```

### 分层职责

| 层 | 职责 | 禁止行为 |
|-----|------|---------|
| **api** | 定义 `XxxReq`/`XxxRes`，`g.Meta` 声明路由、method、tags | 不写业务逻辑 |
| **controller** | 参数转发，调用 service，组装结果 | **禁止直接调用 dao**，**禁止写业务逻辑** |
| **service** | 声明接口 `ISysXxx`，提供注册函数 `RegisterSysXxx()` | **文件只能追加，不能新建** |
| **logic** | 实现 service 接口，执行业务逻辑 | **禁止出现 HTTP 相关代码**，禁止做 `Filter()` 已完成的校验 |
| **dao** | 数据访问基础 CRUD | **`internal/dao/internal/` 禁止手改（gf gen dao 自动生成）** |
| **model** | 数据结构定义：entity/do（生成代码）/input（手写业务DTO） | entity/do 禁止手改 |

## 2.3 API 层（`api/admin/<module>/<module>.go`）

每个接口定义一对 `XxxReq` / `XxxRes` 结构体：

```go
// GET 列表接口
type ListReq struct {
    g.Meta `path:"/xxxModule/list" method:"get" tags:"XXX模块" summary:"获取XXX列表"`
    sysin.XxxListInp
}
type ListRes struct {
    form.PageRes
    List []*sysin.XxxListModel `json:"list" dc:"数据列表"`
}

// POST 编辑接口
type EditReq struct {
    g.Meta `path:"/xxxModule/edit" method:"post" tags:"XXX模块" summary:"新增/编辑XXX"`
    sysin.XxxEditInp
}
type EditRes struct {
    Id int64 `json:"id" dc:"ID"`
}
```

**关键约定**：
- GET 接口用 `method:"get"`，写操作用 `method:"post"`
- 请求体嵌入 `sysin.XxxInp`，响应体嵌入对应的 Model
- 空结构体 `struct{}` 用于无返回数据的接口

## 2.4 Controller 层（`internal/controller/admin/sys/<module>.go`）

**核心原则**：薄控制器，只做参数转发和结果组装。

```go
package sys

import (
    "context"
    "hotgo/api/admin/xxxmodule"
    "hotgo/internal/model/input/sysin"
    "hotgo/internal/service"
)

var XxxModule = cXxxModule{}

type cXxxModule struct{}

// List 列表
func (c *cXxxModule) List(ctx context.Context, req *xxxmodule.ListReq) (res *xxxmodule.ListRes, err error) {
    list, totalCount, err := service.SysXxxModule().List(ctx, &req.XxxListInp)
    if err != nil {
        return
    }
    if list == nil {
        list = []*sysin.XxxListModel{}
    }
    res = new(xxxmodule.ListRes)
    res.List = list
    res.PageRes.Pack(req, totalCount)
    return
}

// Edit 编辑
func (c *cXxxModule) Edit(ctx context.Context, req *xxxmodule.EditReq) (res *xxxmodule.EditRes, err error) {
    id, err := service.SysXxxModule().Edit(ctx, &req.XxxEditInp)
    if err != nil {
        return
    }
    res = &xxxmodule.EditRes{Id: id}
    return
}

// Delete 删除
func (c *cXxxModule) Delete(ctx context.Context, req *xxxmodule.DeleteReq) (res *xxxmodule.DeleteRes, err error) {
    err = service.SysXxxModule().Delete(ctx, &req.XxxDeleteInp)
    return
}
```

**禁止行为**：
- 禁止在 controller 中直接操作 dao
- 禁止在 controller 中包装 error（直接 return 透传）
- 禁止在 controller 中 `fmt.Println` 或 `g.Log().Error` 打印框架已处理的错误

## 2.5 Service 层（`internal/service/sys.go`）

**文件只能追加，不能新建！** 在已有的 `sys.go` 文件中追加接口声明和注册函数：

```go
// ==================== XxxModule ====================

// ISysXxxModule XXX模块服务接口
type ISysXxxModule interface {
    List(ctx context.Context, in *sysin.XxxListInp) (list []*sysin.XxxListModel, totalCount int, err error)
    View(ctx context.Context, in *sysin.XxxViewInp) (res *sysin.XxxViewModel, err error)
    Edit(ctx context.Context, in *sysin.XxxEditInp) (id int64, err error)
    Delete(ctx context.Context, in *sysin.XxxDeleteInp) (err error)
    Status(ctx context.Context, in *sysin.XxxStatusInp) (err error)
    Switch(ctx context.Context, in *sysin.XxxSwitchInp) (err error)
    MaxSort(ctx context.Context, in *sysin.XxxMaxSortInp) (maxSort int, err error)
}

var insSysXxxModule ISysXxxModule

// SysXxxModule 获取 XXX 模块服务实例
func SysXxxModule() ISysXxxModule {
    if insSysXxxModule == nil {
        panic("SysXxxModule 未注册")
    }
    return insSysXxxModule
}

// RegisterSysXxxModule 注册 XXX 模块服务
func RegisterSysXxxModule(i ISysXxxModule) {
    insSysXxxModule = i
}
```

## 2.6 Logic 层（`internal/logic/sys/<module>.go`）

Logic 层是业务逻辑的核心，每个文件遵循统一模式：

```go
package sys

import (
    "context"
    "hotgo/internal/consts"
    "hotgo/internal/dao"
    "hotgo/internal/library/hgorm/handler"
    "hotgo/internal/model/entity"
    "hotgo/internal/model/input/sysin"
    "hotgo/internal/service"
    "github.com/gogf/gf/v2/database/gdb"
    "github.com/gogf/gf/v2/errors/gerror"
    "github.com/gogf/gf/v2/frame/g"
)

type sSysXxxModule struct{}

// init 注册到 service
func init() {
    service.RegisterSysXxxModule(NewSysXxxModule())
}

func NewSysXxxModule() *sSysXxxModule {
    return &sSysXxxModule{}
}

// Model 获取带权限过滤的 ORM 模型
func (s *sSysXxxModule) Model(ctx context.Context, option ...*handler.Option) *gdb.Model {
    return handler.Model(dao.XxxTable.Ctx(ctx), option...)
}

// List 列表查询
func (s *sSysXxxModule) List(ctx context.Context, in *sysin.XxxListInp) (list []*sysin.XxxListModel, totalCount int, err error) {
    mod := s.Model(ctx)
    // 使用 Columns() 引号字段名，禁止硬编码字段名字符串
    if in.Name != "" {
        mod = mod.WhereLike(dao.XxxTable.Columns().Name, "%"+in.Name+"%")
    }
    if in.Status > 0 {
        mod = mod.Where(dao.XxxTable.Columns().Status, in.Status)
    }
    totalCount, err = mod.Count()
    if err != nil || totalCount == 0 {
        return
    }
    err = mod.Page(in.Page, in.PageSize).OrderDesc(dao.XxxTable.Columns().Id).Scan(&list)
    return
}

// Edit 新增/编辑
func (s *sSysXxxModule) Edit(ctx context.Context, in *sysin.XxxEditInp) (id int64, err error) {
    if err = in.Filter(ctx); err != nil {
        return
    }
    // id > 0 为编辑，否则为新增
    if in.Id > 0 {
        _, err = s.Model(ctx).Fields(sysin.XxxUpdateFields{}).WherePri(in.Id).Data(in).Update()
        id = in.Id
    } else {
        _, err = s.Model(ctx).Fields(sysin.XxxInsertFields{}).Data(in).InsertAndGetId()
        id, _ = g.TryConvertInt64(err) // InsertAndGetId 返回 lastInsertId,err
    }
    return
}

// Delete 删除（支持批量）
func (s *sSysXxxModule) Delete(ctx context.Context, in *sysin.XxxDeleteInp) (err error) {
    if len(in.Ids) == 0 {
        return gerror.New("请选择删除项")
    }
    _, err = s.Model(ctx).WhereIn(dao.XxxTable.Columns().Id, in.Ids).Delete()
    return
}
```

**Logic 层关键约定**：
- 结构体名：`sSysXxxModule`（私有）
- `init()` 中调用 `service.RegisterXxx()` 注册
- `Model()` 方法返回带权限过滤的 ORM 模型
- **列名引用必须用 `dao.Table.Columns().FieldName`**，禁止硬编码 `"field_name"`
- `Edit` 方法以 `in.Id > 0` 区分更新/新增，更新用 `XxxUpdateFields`，新增用 `XxxInsertFields`
- Logic 之间通过 `service.Xxx().Method()` 调用，不直接 new
- 命名返回值 `(xxx, err error)`，设置 err 后必须 `return`

## 2.7 Model 层（`internal/model/input/sysin/<module>.go`）

定义所有业务入参、出参和过滤结构：

```go
package sysin

import (
    "context"
    "hotgo/internal/model/entity"
    "hotgo/internal/model/input/form"
    "github.com/gogf/gf/v2/frame/g"
    "github.com/gogf/gf/v2/os/gtime"
)

// ============ 字段白名单 ============

type XxxUpdateFields struct {
    Name   string      `json:"name"`
    Status int         `json:"status"`
    Sort   int         `json:"sort"`
    Remark string      `json:"remark"`
}

type XxxInsertFields struct {
    XxxUpdateFields
    CreatedBy int64 `json:"createdBy"`
}

// ============ 编辑 ============

type XxxEditInp struct {
    entity.XxxTable                 // 嵌入 entity
}

func (in *XxxEditInp) Filter(ctx context.Context) (err error) {
    if in.Name == "" {
        return gerror.New("名称为空")
    }
    // 更多校验...
    return
}

// ============ 列表 ============

type XxxListInp struct {
    form.PageInp
    Name   string `json:"name" dc:"名称"`
    Status int    `json:"status" dc:"状态"`
}

type XxxListModel struct {
    Id     int64       `json:"id"`
    Name   string      `json:"name"`
    Status int         `json:"status"`
    CreatedAt *gtime.Time `json:"createdAt"`
}

// ============ 其他操作 ============

type XxxDeleteInp struct {
    Ids []int64 `json:"ids" v:"required#请选择删除项"`
}

type XxxStatusInp struct {
    Id     int64 `json:"id" v:"required#ID不能为空"`
    Status int   `json:"status" v:"required#状态不能为空"`
}

type XxxSwitchInp struct {
    Id   int64  `json:"id" v:"required#ID不能为空"`
    Key  string `json:"key" v:"required#开关字段不能为空"`
    Value int   `json:"value"`
}

type XxxMaxSortInp struct {
    Id int64 `json:"id"`
}

type XxxViewInp struct {
    Id int64 `json:"id" v:"required#ID不能为空"`
}

type XxxViewModel struct {
    entity.XxxTable
}
```

**Model 结构类型速查**：

| 类型 | 用途 |
|------|------|
| `XxxUpdateFields` | 更新时字段白名单 |
| `XxxInsertFields` | 新增时字段白名单 |
| `XxxEditInp` | 编辑入参（嵌入 entity，含 `Filter()` 校验） |
| `XxxListInp` | 列表查询入参（嵌入 `form.PageInp`） |
| `XxxListModel` | 列表返回字段 |
| `XxxViewInp` / `XxxViewModel` | 详情入参/出参 |
| `XxxDeleteInp` | 删除入参（通常含 `Ids []int64`） |
| `XxxStatusInp` | 状态切换入参 |
| `XxxSwitchInp` | 开关切换入参 |

|------|--------|------|
| `/xxx/list` | GET | 分页列表 |
| `/xxx/view` | GET | 详情 |
| `/xxx/edit` | POST | 新增/编辑（`id>0` 为编辑） |
| `/xxx/delete` | POST | 删除（支持批量，传 `ids`） |
| `/xxx/status` | POST | 启用/禁用状态切换 |
| `/xxx/switch` | POST | 开关字段切换 |
| `/xxx/maxSort` | GET | 获取最大排序值 |
| `/xxx/export` | GET | 导出列表 |

按实际需求裁剪，不强制全部实现。
