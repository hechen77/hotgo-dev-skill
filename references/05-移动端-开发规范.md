## 5.1 环境与项目创建

### 环境要求

| 工具 | 版本要求 | 验证命令 |
|------|---------|---------|
| Flutter SDK | >= 3.11.5 | `flutter --version` |
| Dart SDK | >= 3.11.5 | `dart --version` |
| Android Studio | 最新稳定版 | — |
| JDK | >= 17 | `java -version` |
| VS Code / Android Studio | 最新版 | — |

### 创建项目

```bash
# 1. 创建 Flutter 项目
flutter create --org com.yourcompany --project-name your_app_name .

# 2. 进入项目目录
cd your_app_name

# 3. 安装依赖
flutter pub get

# 4. 运行到已连接的 Android 设备
flutter run

# 5. 运行到指定设备
flutter devices                # 列出可用设备
flutter run -d <device_id>     # 指定设备

# 6. 构建 APK
flutter build apk              # 默认 APK
flutter build apk --split-per-abi  # 按 CPU 架构分包
flutter build appbundle        # Google Play 格式

# 7. 构建 iOS
flutter build ios
```

### Flutter 项目初始骨架

```
your_app/
├── android/           # Android 原生工程
```

### 每个依赖的作用详解

| 依赖 | 为什么需要 | 如果没有会怎样 |
|------|----------|-------------|
| `get` | 路由导航、GetBuilder 状态管理、GetxController 生命周期、依赖注入(`Get.put`/`Get.find`) | 需要手动管理路由栈、用 Provider/Riverpod 替代状态管理、手动处理 Controller 生命周期 |
| `dio` | HTTP 拦截器、请求/响应转换、超时控制、BaseUrl 管理 | 需要用 `http` 包手动写拦截器逻辑 |
| `flutter_secure_storage` | Token 等敏感数据的加密存储（底层用 Android Keystore / iOS Keychain） | Token 存 SharedPreferences 有安全隐患 |
| `flutter_foreground_task` | 保持 WebSocket 长连接在 App 后台时不被系统杀死 | App 切后台后 WS 断开，实时推送失效 |
| `flutter_local_notifications` | 收到 WS 消息后在通知栏弹出提醒 | 无法在后台通知用户有新消息 |
| `tdesign_flutter` | 提供按钮、对话框、输入框、标签页等移动端 UI 组件 | 需要自行封装或引入 Material/Cupertino 组件 |
| `encrypt` | 登录密码 AES 加密传输 | 密码明文传输 |
| `web_socket_channel` | 与服务端建立 WebSocket 长连接 | 无法实时接收推送 |
| `logger` | 结构化日志输出（按级别过滤、格式化） | 用 `print`/`debugPrint` 难以区分日志级别 |
| `connectivity_plus` | 请求前检测网络状态 | 无网络时发起请求会有较长等待超时 |
| `device_info_plus` | 获取 Android ID 作为设备唯一标识 | 无法精确识别设备 |
| `permission_handler` | 动态请求通知、存储等运行时权限 | Android 13+ 通知权限默认关闭 |
| `cached_network_image` | 列表中的网络图片缓存到磁盘 | 每次滚动都重新加载图片，流量浪费且卡顿 |

### 5.3.4 android/key.properties（Release 签名）

```properties
storePassword=你的store密码
keyPassword=你的key密码
keyAlias=你的key别名
storeFile=your_keystore.jks
```

**生成签名文件**：
```bash
keytool -genkey -v -keystore your_keystore.jks \
  -keyalg RSA -keysize 2048 -validity 10000 \
  -alias your_alias
```

把 `.jks` 文件放到 `android/` 目录下，命名为 `key.properties` 中指定的文件名。

## 5.6 入口文件 main.dart

```dart

  @override
```

```dart
      await storage.read(key: key);

      await storage.delete(key: key);
```

**为什么用 `flutter_secure_storage` 而不是 `shared_preferences`？**
- Token 是敏感信息，`shared_preferences` 明文存储在 XML 文件中
- `flutter_secure_storage` 底层用 Android Keystore / iOS Keychain 加密存储
- `shared_preferences` 可用于非敏感数据（如用户偏好设置），但项目中已统一用 `flutter_secure_storage`

```dart
  final String message;
  BusinessException(this.code, this.message);
  @override
  String toString() => 'BusinessException($code): $message';

class ApiResponse<T> {

  final String message;
  final T? data;
  ApiResponse({required this.code, required this.message, this.data});

  factory ApiResponse.fromMap(Map<String, dynamic> map) {
    return ApiResponse(

      message: map['message'] ?? '',

/// ⭐ 网络请求管理器（单例）
class HttpManager {
  static HttpManager? _instance;
  late Dio _dio;
  final Connectivity _connectivity = Connectivity();

  HttpManager._internal() {
    _initDio();
    _initInterceptors();
  }

  static HttpManager get instance {
    _instance ??= HttpManager._internal();
    return _instance!;
  }

  void _initDio() {
    _dio = Dio(BaseOptions(
      baseUrl: AppConfig().baseUrl(),
      connectTimeout: const Duration(seconds: 30),
      receiveTimeout: const Duration(seconds: 30),
      headers: {
        'Content-Type': 'application/json;charset=utf-8',
        'Accept': 'application/json',
      },
      responseType: ResponseType.json,
    ));
  }

  void _initInterceptors() {
    // Debug 模式打印日志
    if (kDebugMode) {
      _dio.interceptors.add(LogInterceptor(requestBody: true, responseBody: true));
    }

    // ⭐ 核心拦截器
    _dio.interceptors.add(InterceptorsWrapper(
      onRequest: (options, handler) async {
        // 自动注入 Token
        String? token = Get.find<AuthService>().token;
        options.headers['Authorization'] = token ?? null;
        options.headers['authclient'] = "platform";
        handler.next(options);
      },
      onResponse: (response, handler) => handler.next(response),
      onError: (error, handler) {
        AppLogger.d("接口出错: ${error.requestOptions.path}");
        handler.reject(DioException(
          requestOptions: error.requestOptions,
          error: _handleError(error),
          type: error.type,
          response: error.response,
        ));
      },
    ));
  }

  String _handleError(DioException error) {
    switch (error.type) {
      case DioExceptionType.connectionTimeout:  return '网络连接超时，请检查网络';
      case DioExceptionType.sendTimeout:         return '请求发送超时，请稍后重试';
      case DioExceptionType.receiveTimeout:      return '响应接收超时，请稍后重试';
      case DioExceptionType.badResponse:
        final statusCode = error.response?.statusCode ?? 0;
        if (statusCode == 61)  return '登录状态过期，请重新登录';
        if (statusCode == 403) return '权限不足';
        if (statusCode == 404) return '请求接口不存在';
        if (statusCode == 500) return '服务器内部错误';
        return '请求失败（$statusCode）';
      case DioExceptionType.connectionError: return '网络连接错误，请检查网络';
      case DioExceptionType.cancel:          return '请求已取消';
      default: return '未知错误：${error.message ?? '请稍后重试'}';
    }
  }

  /// ⭐ 通用请求方法
  Future<ApiResponse> request({
    required String path,
    required HttpMethod method,
    Map<String, dynamic>? params,
    Map<String, dynamic>? queryParams,
    bool showLoading = true,
  }) async {
    if (showLoading) LoadingController.instance.show();
    try {
      // 网络检查
      final hasNetwork = await _connectivity.checkConnectivity()
          .then((r) => r != ConnectivityResult.none);
      if (!hasNetwork) {
        throw DioException(
          requestOptions: RequestOptions(path: path),
          error: '当前无网络连接',
          type: DioExceptionType.connectionError,
        );
      }

      Response response;
      switch (method) {
        case HttpMethod.get:
          response = await _dio.get(path, queryParameters: queryParams ?? params);
        case HttpMethod.post:
          response = await _dio.post(path, data: params, queryParameters: queryParams);
        case HttpMethod.put:
          response = await _dio.put(path, data: params, queryParameters: queryParams);
        case HttpMethod.delete:
          response = await _dio.delete(path, data: params, queryParameters: queryParams);
      }

      if (response.data is! Map<String, dynamic>) {
        throw DioException(
          requestOptions: response.requestOptions,
          error: '返回数据格式异常',
          type: DioExceptionType.badResponse,
        );
      }

      final apiResponse = ApiResponse.fromMap(response.data);

      // ⭐ code != 0 → 业务异常
      if (apiResponse.code != 0) {
        if (apiResponse.code == 61) Get.find<AuthService>().logout(false);
        throw BusinessException(apiResponse.code, apiResponse.message);
      }

      return apiResponse;
    } finally {
      if (showLoading) LoadingController.instance.hide();
    }
  }
}
```

```dart
  final int pageSize;

      pageCount: json['pageCount'] ?? 0,
```

### Service 层规范总结

| 规范 | 说明 |
|------|------|
| 继承 `GetxService` | GetX 全局服务基类，常驻内存 |
| `init()` 返回 `Future<Self>` | 配合 `Get.putAsync()` 异步初始化 |
| `init()` 内必须有 try-catch | 启动阶段任何一个 Service 报错都不应阻止 App 启动 |
| 必须有 `onClose()` | 即使只是打印日志 |
| 通过 `Get.find<XxxService>()` 获取 | 不直接 new，不通过构造函数传参 |
| 初始化顺序固定 | AuthService → ConfigService → UserService |

### 5.12.1 列表页标准模式（三方法）

**Controller**：

```dart
import 'package:get/get.dart';
import 'package:your_app/api/models/general_model.dart';
import 'package:your_app/api/article/article.dart';
import 'package:your_app/api/article/model/Article.dart';
import 'package:your_app/utils/app_logger.dart';

class ArticleListPageController extends GetxController {
  List<Article> list = [];
  ListQueryParams params = ListQueryParams(page: 1, pageSize: 20);
  bool isLoading = false;
  bool hasMore = true;

  @override
  void onInit() {
    super.onInit();
    pageRefresh();  // 进入页面自动加载第一页
  }

  Future<void> loadList({bool isRefresh = false}) async {
    if (isLoading) return;               // ⚠️ 防止重复请求
    isLoading = true;
    update();
    try {
      if (isRefresh) {
        params.page = 1;
        list.clear();
      }
      final res = await ArticleApi.getList(params);
      list.addAll(res.list);
      hasMore = params.page < res.pageCount;
      isLoading = false;
      update();
    } catch (e, s) {
      AppLogger.e("列表加载失败", error: e, stackTrace: s);
      isLoading = false;                 // ⚠️ 必须在 catch 中重置
      update();                          // ⚠️ 必须在 catch 中更新 UI
    }
  }

  Future<void> pageRefresh() async => await loadList(isRefresh: true);

  Future<void> loadMore() async {
    if (!hasMore || isLoading) return;
    params.page++;
    await loadList();
  }

  @override
  void onClose() { super.onClose(); }    // ⚠️ 必须有
}
```

**View**：

```dart
class ArticleListPage extends StatefulWidget {
  const ArticleListPage({super.key});
  @override
  State<ArticleListPage> createState() => _ArticleListPageState();
}

class _ArticleListPageState extends State<ArticleListPage> {
  @override
  Widget build(BuildContext context) {
    return GetBuilder(
      init: ArticleListPageController(),
      builder: (controller) {
        return Scaffold(
          appBar: AppBar(title: const Text("标题")),
          body: RefreshIndicator(
            onRefresh: controller.pageRefresh,
            child: _buildList(controller),
          ),
        );
      },
    );
  }

  Widget _buildList(ArticleListPageController controller) {
    // 首次加载中
    if (controller.isLoading && controller.list.isEmpty) {
      return const Center(child: CircularProgressIndicator());
    }
    // 空数据
    if (controller.list.isEmpty) {
      return const Center(child: Text("暂无数据"));
    }
    // 列表
    return NotificationListener<ScrollNotification>(
      onNotification: (notification) {
        // ⭐ 滚动到底部自动加载更多
        if (notification.metrics.pixels >=
            notification.metrics.maxScrollExtent - 100) {
          controller.loadMore();
        }
        return false;
      },
      child: ListView.builder(
        itemCount: controller.list.length + (controller.hasMore ? 1 : 0),
        itemBuilder: (context, index) {
          if (index == controller.list.length) {
            return const Center(child: CircularProgressIndicator());
          }
          return _buildItem(controller.list[index]);
        },
      ),
    );
  }

  Widget _buildItem(Article item) {
    return ListTile(
      title: Text(item.title ?? ''),
      subtitle: Text(item.summary ?? ''),
      onTap: () => Routes.toDetail(item.id!),
    );
  }
}
```

## 5.13 Controller 规范

| 规范 | 说明 |
|------|------|
| 继承 `GetxController` | 或 `GetxController` 的变体 |
| `onInit()` 中发起首次数据请求 | 进入页面自动加载 |
| 所有 API 调用包裹 try-catch | 绝对不允许裸调 API |
| catch 中必须重置 `isLoading` | 否则 UI 永远显示加载中 |
| catch 中必须调用 `update()` | 否则 UI 不刷新 |
| 必须有 `onClose()` | GetX 生命周期方法，释放资源 |
| 如有 `TextEditingController` | 在 `onClose()` 中 `dispose()` |
| 用 `update()` 触发 UI 刷新 | GetBuilder 模式 |
| 用 `Get.find<XxxService>()` | 获取全局服务，不传参 |

**完整 Controller 模板**：


## 5.16 前台任务守护

`lib/utils/foreground/foreground_task_manager.dart`：

**为什么需要？** Android 系统会在 App 切后台一段时间后杀死进程，导致 WebSocket 断开，无法接收实时推送。前台服务（Foreground Service）通过常驻通知告知系统"我是重要进程，别杀我"。

**工作流程**：
1. App 进入首页 → `ForegroundTaskManager.instance.start()` 启动前台服务
2. 后台 Isolate 每隔 60 秒 `onRepeatEvent` 检查 WS 状态 → 断开则重连
3. 用户登出 → `ForegroundTaskManager.instance.stop()` 停止

**⚠️ 关键注意事项**：
- 后台 Isolate 中**不能访问 GetX 单例**和 `FlutterSecureStorage`
- WebSocket 重连逻辑需要通过 `WebSocketManager.instance`（静态单例）绕过 Isolate 隔离
- 在 `AndroidManifest.xml` 中必须声明 `foregroundServiceType`

## 5.18 全局 Loading

`lib/components/loading_controller/loading_controller.dart`：

**设计**：引用计数模式。`show()` 计数 +1，`hide()` 计数 -1，计数 > 0 时显示遮罩。这确保多个并发请求不会提前关闭 Loading。

```dart
class LoadingController extends ChangeNotifier {
  LoadingController._();
  static final LoadingController instance = LoadingController._();

  int _count = 0;
  bool get isLoading => _count > 0;

  void show() { _count++; if (_count == 1) notifyListeners(); }
  void hide() { if (_count > 0) { _count--; if (_count == 0) notifyListeners(); } }
}
```
