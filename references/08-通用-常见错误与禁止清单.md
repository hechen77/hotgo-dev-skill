## 8.1 后端（Go + GoFrame）

### ❌ 严格禁止

| 禁止行为 | 正确做法 |
|---------|---------|
| controller 直接调用 dao | controller → service → logic → dao |
| 硬编码字段名字符串 `"create_time"` | 使用 `dao.Table.Columns().CreatedAt` |
| 手改 `entity/`、`do/`、`dao/internal/` | `gf gen dao` 自动生成 |
| logic 重复做 `Filter()` 已完成的校验 | 校验只在 `sysin.Filter(ctx)` 中完成 |
| 在 `internal/service/sys.go` 之外新建 service 文件 | 统一追加到 `sys.go` |
| 插件 import 另一个插件的包 | 插件间通过主模块 service 通信 |
| 插件路由硬编码路径前缀 | 用 `addons.RouterPrefix()` |
| controller 层手动包装响应 | 交给 `middleware.ResponseHandler` |
| `fmt.Println` 替代日志 | 用 `g.Log().Infof(ctx, ...)` |
| controller 层打印 framework 已记录的 error | 直接 `return` 透传 |
| 修改 `internal/library/` 下的基础库 | 在 `utility/` 添加工具或提 issue |

### ⚠️ 常见错误模式

**1. 设置 err 后未 return**
```go
// ❌ 错误
func (s *sXxx) Query(ctx context.Context) (res *Model, err error) {
    err = gerror.New("没有权限")  // 设了 err 但没 return
    return dao.Xxx.Ctx(ctx).One()  // 继续执行，可能覆盖 err 或 panic
}

// ✅ 正确
func (s *sXxx) Query(ctx context.Context) (res *Model, err error) {
    if err = checkAuth(ctx); err != nil {
        return  // 必须立即 return
    }
    return dao.Xxx.Ctx(ctx).One()
}
```

**2. 用 err1/err2 绕过命名返回值 err**
```go
// ❌ 错误
func (s *sXxx) Save(ctx context.Context) (err error) {
    err1 := operation1()  // 不应该用 err1
    err2 := operation2()  // 不应该用 err2
    return nil  // 丢失了 err1 和 err2
}

// ✅ 正确
func (s *sXxx) Save(ctx context.Context) (err error) {
    if err = operation1(); err != nil {
        return
    }
    if err = operation2(); err != nil {
        return
    }
    return
}
```

**3. 列表为空时访问首元素**
```go
// ❌ 错误
func (s *sXxx) Process(ctx context.Context, in *sysin.Inp) (err error) {
    item := in.List[0]  // len(in.List) == 0 时 panic
}

// ✅ 正确
func (s *sXxx) Process(ctx context.Context, in *sysin.Inp) (err error) {
    if len(in.List) == 0 {
        return gerror.New("列表不能为空")
    }
    item := in.List[0]
}
```

**4. 错误消息 copy-paste 不改上下文**
```go
// ❌ 错误
return gerror.New("创建失败，请稍后重试")   // 在"更新"方法里用了"创建"的文案

// ✅ 正确
return gerror.New("更新XXX失败，请稍后重试")
```

## 8.2 Web 前端（Vue 3 + Naive UI）

### ❌ 严格禁止

| 禁止行为 | 正确做法 |
|---------|---------|
| 直接使用 axios | 用 `http.request` |
| 硬编码字典值文本 `v-if="row.status === 1"` | 用 `dict.getLabel('XxxStatusOptions', row.status)` |
| 引入 Naive UI 以外的 UI 框架 | 只用 Naive UI |
| 内联 `style="color: red"` | 用 Naive UI props 或 class |
| 源码里硬编码菜单路由路径 | 路由由后端菜单数据驱动 |
| 在 `internal/service/sys.go` 之外新建 service 文件 | 同文件追加 |

### ⚠️ 常见错误模式

**1. dict.getOptions 不需要 computed**
```vue
<!-- ✅ 正确：字典数据本身已是响应式，直接使用即可 -->
<n-select :options="dict.getOptions('XxxStatusOptions')" />
```

**2. 忘记在 model.ts 中预加载字典**
```ts
// ❌ 错误：跳过了 loadOptions
export function loadOptions() {
    // 空或忘记调用
}

// ✅ 正确
export function loadOptions() {
    dict.loadOptions(['XxxStatusOptions', 'sys_normal_disable']);
}
```

## 8.3 Flutter 移动端

### ❌ 严格禁止

| 禁止行为 | 正确做法 |
|---------|---------|
| `debugPrint()` / `print()` / `Logger()` | `AppLogger.d/i/e()` |
| 散落的硬编码 URL / 常量 | 统一放 `AppConfig` 中 |
| Controller 无 `onClose()` | 所有 Controller 必须有 `onClose()` |
| API 调用不包裹 try-catch | 所有 API 调用必须有 try-catch |

### ⚠️ 常见错误模式

**1. catch 中忘记重置 isLoading**
```dart
// ❌ 错误
try {
    final res = await XxxApi.getList(params);
    isLoading = false; update();
} catch (e) {
    AppLogger.e("error", error: e);
    // 忘记 isLoading = false; update();
}

// ✅ 正确
try {
    final res = await XxxApi.getList(params);
    isLoading = false; update();
} catch (e, s) {
    AppLogger.e("error", error: e, stackTrace: s);
    isLoading = false; update();  // ⚠️ catch 中也要重置
}
```

**2. Service 初始化无 try-catch 导致启动卡死**
```dart
// ❌ 错误
Future<ConfigService> init() async {
    appConfig = await AppConfigApi.getAppConfig();  // 异常会导致启动卡死
    return this;
}

// ✅ 正确
Future<ConfigService> init() async {
    try {
        appConfig = await AppConfigApi.getAppConfig();
    } catch (e, s) {
        AppLogger.e('初始化失败', error: e, stackTrace: s);
    }
    return this;
}
```

**3. 忘记 dispose TextEditingController**
```dart
// ❌ 错误
class LoginPageController extends GetxController {
    final mobileController = TextEditingController();
    // 忘记 onClose() 中 dispose
}

// ✅ 正确
class LoginPageController extends GetxController {
    final mobileController = TextEditingController();
    final passwordController = TextEditingController();

    @override
    void onClose() {
        mobileController.dispose();
        passwordController.dispose();
        super.onClose();
    }
}
```

**4. Isolate 隔离问题（后台任务）**
```dart
// ❌ 错误：后台 Isolate 访问 GetX 单例
void backgroundTask() {
    var token = Get.find<AuthService>().token;  // Isolate 隔离，获取不到
}

// ✅ 正确：通过 FlutterSecureStorage 直接读取
void backgroundTask() async {
    const storage = FlutterSecureStorage();
    var token = await storage.read(key: 'token');
}
```

## 8.4 Flutter 桌面端

### ❌ 不正确的做法

| 错误 | 正确 |
|------|------|
| 关闭按钮退出程序 | 隐藏到托盘 `windowManager.hide()` |
| 登录和主界面同窗口大小 | 登录锁 320×480，主界面 1200×800 |
| `onWindowClose` 中 `await windowManager.close()` | `await windowManager.hide()` |
| 忘记在 `dispose` 中 `removeListener` | 配对 `addListener` / `removeListener` |

## 8.5 小程序（Uni-app）

### ❌ 常见错误

| 错误 | 正确 |
|------|------|
| 微信 API 直接调用 | 通过 `uni.` API 统一调用 |
| 页面未在 pages.json 注册 | 每次新增页面必须注册 |
| 忘记配置 `mp-weixin.appid` | manifest.json 中正确填写 |
| `scroll-view` 不设高度 | 必须明确设置高度 |
| rpx 和 px 混用 | 统一使用 rpx（750rpx = 屏幕宽度） |

## 8.6 跨端通用原则

1. **修改代码前先读 `PROJECT_CONTEXT.md`**，了解之前的优化和已知问题
2. **不要重复之前已修过的问题**（见 PROJECT_CONTEXT.md 的优化列表）
3. **分层调用**：禁止跨层调用，严格遵守每层的职责边界
4. **错误处理**：每层对错误的处理方式不同，不要层内重复包装 error
5. **日志统一**：使用各端统一的日志工具，禁止裸 `print`
6. **常量集中**：不散落魔法数字和硬编码字符串
7. **代码去重**：重复 3 次以上的代码提取为工具方法
8. **先读再改**：在现有风格基础上修改，不自创新的代码模式
