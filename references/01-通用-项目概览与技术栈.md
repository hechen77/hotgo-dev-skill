## 1.1 技术栈总览

| 层级 | 技术 | 版本/说明 |
|------|------|----------|
| **后端语言** | Go | `go module` 名 `hotgo` |
| **后端框架** | GoFrame v2 | gdb、ghttp、gcron、gerror、gctx、gjson、gvar |
| **数据库** | PostgreSQL / MySQL | 根据项目选择，表前缀 `hg_` |
| **缓存** | Redis | 统一通过 `cache.Instance()` 操作 |
| **权限** | Casbin | RBAC 角色权限控制 |
| **认证** | JWT | 无状态 Token 认证 |
| **消息队列** | Redis/Disk/Kafka/RocketMQ | 支持多种驱动 |
| **Web后台前端** | Vue 3 + Vite + TypeScript | Composition API (`<script setup>`) |
| **Web UI 组件库** | Naive UI | **唯一 UI 库**，禁止引入其他 UI 库 |
| **Web 状态管理** | Pinia | Vue 3 官方推荐 |
| **移动端 App** | Flutter + Dart | SDK ^3.11.5 |
| **移动端状态管理** | GetX | GetBuilder + GetxController |
| **移动端 UI 组件库** | TDesign Flutter | 腾讯开源组件库 |
| **桌面端** | Flutter Desktop | 支持 Windows / macOS / Linux |
| **桌面端 UI 框架** | Forui + TDesign Flutter | 现代化桌面风格 |
| **小程序** | Uni-app (Vue 3) | 支持微信小程序等多端 |
| **小程序 UI 组件库** | uv-ui | Uni-app 生态组件库 |

## 1.2 项目结构（Monorepo）

```
项目根目录/
├── server/                  # Go 后端服务（基于 HotGo）
│   ├── addons/              # 插件模块（业务模块）
│   ├── api/                 # HTTP API 契约定义
│   ├── internal/            # 内部核心代码
│   ├── manifest/            # 配置、部署文件
│   ├── resource/            # 静态资源和代码生成模板
│   ├── utility/             # 通用工具库
│   ├── go.mod
│   └── main.go              # 程序入口
├── web/                     # Web 管理后台（Vue 3 + Naive UI）
│   ├── src/
│   │   ├── api/             # HTTP 请求封装
│   │   ├── components/      # 通用组件
│   │   ├── hooks/           # 组合式函数
│   │   ├── router/          # 路由 + 权限守卫
│   │   ├── store/           # Pinia 状态管理
│   │   ├── utils/           # 工具类
│   │   └── views/           # 页面组件
│   └── package.json
├── mobile/                  # 移动端 App（Flutter）
│   └── lib/
│       ├── api/             # API 客户端模块
│       ├── pages/           # 页面
│       ├── routers/         # 路由定义
│       ├── service/         # 全局服务
│       ├── components/      # 共享组件
│       └── utils/           # 工具类
├── desktop/                 # 桌面端（Flutter Desktop）
│   └── lib/
│       ├── api/
│       ├── pages/
│       ├── service/         # 窗口管理、系统托盘等
│       └── components/      # 标题栏、侧边栏等桌面组件
└── webview/                 # 小程序 / H5（Uni-app）
    ├── pages/               # 小程序页面
    ├── uni_modules/         # Uni-app 模块（uv-ui）
    ├── pages.json           # 页面路由配置
    └── manifest.json        # 应用配置
```

## 1.3 四大路由模块

HotGo 后端设计了四大路由模块，分别服务于不同的客户端类型：

| 模块 | URL 前缀 | 认证方式 | 用户角色 |
|------|---------|---------|---------|
| `admin` | `/admin` | JWT + Casbin RBAC | 后台管理员 |
| `api` | `/api` | JWT（部分路径豁免） | 外部脚本/设备/桌面端 |
| `home` | `/home` | JWT | 移动端 App 用户 |
| `websocket` | `/socket` | WebSocket 握手鉴权 | 所有在线用户 |

## 1.4 请求中间件链

每个 HTTP 请求经过以下中间件链：

```
1. middleware.Ctx          → 初始化上下文、i18n、链路追踪
2. middleware.CORS         → 跨域处理
3. middleware.Blacklist    → IP 黑名单检查
4. middleware.DemoLimit    → 演示模式限制
5. middleware.Addon        → 解析 URL 确定插件模块名
6. middleware.PreFilter    → 请求输入预处理
7. middleware.DeliverUserContext → JWT 解析 + 按 App 类型绑定用户
8. middleware.AdminAuth    → Casbin RBAC 权限检查（admin 模块）
9. Controller → Logic → DAO → DB
10. middleware.ResponseHandler → 统一响应格式化
```

## 1.5 多 Addon 协作示例

```
移动端App ──查询请求──→ Go Backend ──WebSocket通知──→ 数据采集端
      ↑                         │                         │
      └── WebSocket 通知 ←─────┘ ←── HTTP API 上报 ←─────┘
```

**多 Addon 协作模式**：
- `user_center` addon：处理用户侧（登录、鉴权、权限校验、发 WS 通知）
- `data_collector` addon：处理采集侧（设备注册、接收上报数据、入库、发 WS 反馈）
- 多个 addon 之间通过主模块 Service 接口通信，不直接互相 import
