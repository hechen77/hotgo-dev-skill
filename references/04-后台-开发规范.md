## 4.1 环境与项目创建

### 环境要求

| 工具 | 版本 | 验证命令 |
|------|------|---------|
| Node.js | >= 20.0.0 | `node -v` |
| pnpm | 最新版 | `pnpm -v` |

```bash
# 安装 pnpm
npm install -g pnpm
```

### 项目创建（从 HotGo 模板开始）

HotGo 的 Web 前端是基于其官方模板的，实际项目不需要从零创建，而是从 HotGo 仓库中获取 `web/` 目录。以下命令展示全新环境下的拉取和初始化流程：

```bash
# 1. 克隆 HotGo 仓库
git clone https://github.com/bufanyun/hotgo.git
cd hotgo

# 2. 进入 web 目录
cd web

# 3. 安装依赖
pnpm install

# 4. 启动开发服务器
pnpm run dev          # 默认 8001 端口，代理 /admin → localhost:8000

# 5. 构建生产版本
pnpm run build        # 输出到 dist/

# 6. 预览构建产物
pnpm run preview
```

### 关键依赖说明

| 依赖 | 为什么需要 | 如果不装 |
|------|----------|---------|
| `naive-ui` | **唯一合法的 UI 组件库**，提供按钮/表格/表单/弹窗/选择器等全部组件 | 没有 UI，所有组件需手写 |
| `vue-router` | Hash 模式路由，按权限动态加载菜单 | 无法做页面导航 |
| `pinia` | 用户状态、字典数据、应用配置的集中管理 | 状态散落，跨组件传参困难 |
| `axios` | HTTP 客户端（项目中通过 `@/utils/http/axios` 封装使用，不直接用） | 无法请求后端 API |
| `vue-i18n` | 多语言支持（简中/繁中/英文） | 无法国际化 |
| `lodash-es` | `cloneDeep` 深拷贝（编辑表单回填时用） | 数据污染 |
| `date-fns` | 日期格式化（`format(new Date(), 'yyyy-MM-dd')`） | 需要手写日期处理 |
| `tailwindcss` | 原子化 CSS 辅助 | 全部手写 CSS |
| `husky` + `lint-staged` | Git 提交前自动 ESLint 修复 | 可能提交有问题的代码 |

## 4.3 Vite 配置详解

`vite.config.ts`：

```ts
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import { resolve } from 'path'

export default defineConfig({
  plugins: [
    vue(),
    // 其他插件从 build/vite/plugin.ts 导入
  ],
  resolve: {
    alias: {
      '@': resolve(__dirname, 'src'),    // ⭐ @ 指向 src/
    }
  },
  server: {
    port: 8001,                           // 开发服务器端口
    proxy: {
      '/admin': {
        target: 'http://localhost:8000',  // ⭐ 代理后端 API
        changeOrigin: true,
      },
      '/api': {
        target: 'http://localhost:8000',
        changeOrigin: true,
      },
      '/home': {
        target: 'http://localhost:8000',
        changeOrigin: true,
      },
      '/socket': {
        target: 'http://localhost:8000',  // WebSocket 代理
        ws: true,
        changeOrigin: true,
      }
    }
  },
  build: {
    outDir: 'dist',
    rollupOptions: {
      output: {
        manualChunks: {
          'naive-ui': ['naive-ui'],       // 拆分包：UI 库独立
          'vendor': ['vue', 'vue-router', 'pinia'],
        }
      }
    }
  }
})
```

### Vite 代理原理

```
浏览器 → localhost:8001/admin/member/list
         → Vite Proxy → localhost:8000/admin/member/list

浏览器 → localhost:8001/socket
         → Vite Proxy (WS) → localhost:8000/socket
```

这样前端开发时无需处理跨域问题。

## 4.6 API 封装层

### HTTP 封装

`utils/http/axios.ts`：

```ts
import axios from 'axios'
import { useUserStoreWidthOut } from '@/store/modules/user'

// ⭐ 创建 Axios 实例
const service = axios.create({
  baseURL: import.meta.env.VITE_GLOB_API_URL,
  timeout: 30000,
})

// ⭐ 请求拦截器：自动注入 Token
service.interceptors.request.use((config) => {
  const userStore = useUserStoreWidthOut()
  if (userStore.token) {
    config.headers.Authorization = userStore.token
  }
  return config
})

// ⭐ 响应拦截器：统一错误处理
service.interceptors.response.use(
  (response) => {
    const { code, message } = response.data
    if (code === 0) {
      return response.data         // 成功：返回 data
    }
    if (code === 61) {
      // Token 过期 → 跳转登录
      const userStore = useUserStoreWidthOut()
      userStore.logout()
    }
    return Promise.reject(new Error(message))
  },
  (error) => Promise.reject(error)
)

// ⭐ 统一请求方法
export const http = {
  request(config) {
    return service.request(config)
  },
  get(url, params) {
    return service.get(url, { params })
  },
  post(url, params) {
    return service.post(url, params)
  }
}

// ⭐ 文件导出
export function jumpExport(url, params) {
  const token = useUserStoreWidthOut().token
  window.open(`${url}?${new URLSearchParams(params)}&authorization=${token}`)
}
```

### model.ts 结构速查

| 导出 | 类型 | 用途 |
|------|------|------|
| `class State` | class | 与后端 entity 字段对应，提供默认值 |
| `newState()` | function | 深拷贝工厂函数（编辑表单回填用） |
| `rules` | object | Naive UI 表单校验规则 |
| `schemas` | `FormSchema[]` | 搜索栏配置（字段 → 组件 → 选项） |
| `columns` | `BasicColumn[]` | 表格列定义（宽度、渲染函数、权限） |
| `loadOptions()` | function | 预加载字典数据（在列表页 setup 中调用） |

## 4.8 列表页标准写法

`views/article/index.vue`：

```vue
<template>
  <BasicTable
    title="文章管理"
    :columns="columns"
    :schemas="schemas"
    :api="api"
    :showAction="true"
    :actionWidth="200"
    @add="handleAdd"
    @edit="handleEdit"
    @delete="handleDelete"
    @status="handleStatus"
    @switch="handleSwitch"
    @export="handleExport"
  />
  <edit v-model:visible="editVisible" v-model:id="editId" @reload="reloadTable" />
</template>

<script lang="ts" setup>
import { ref } from 'vue'
import BasicTable from '@/components/Table/src/BasicTable.vue'
import edit from './edit.vue'
import { columns, schemas, loadOptions } from './model'
import { List, Delete, Status, Switch, Export } from '@/api/article'

const tableRef = ref()
const editVisible = ref(false)
const editId = ref(0)

const api = {
  list: List,
  delete: Delete,
  status: Status,
  switch: Switch,
  export: Export,
}

loadOptions()  // ⚠️ 必须在 setup 中调用

function handleAdd() {
  editId.value = 0
  editVisible.value = true
}

function handleEdit(id: number) {
  editId.value = id
  editVisible.value = true
}

function reloadTable() {
  tableRef.value?.reload()
}
</script>
```

## 4.11 权限控制

### 主模块权限

```ts
import { usePermission } from '@/hooks/web/usePermission'
const { hasPermission } = usePermission()

// 模板中隐藏按钮
<template>
  <n-button v-if="hasPermission(['/article/edit'])" @click="handleEdit">编辑</n-button>
</template>

// columns 中禁用操作
{
  title: '操作',
  key: 'action',
  render(row) {
    return h('div', [
      h('n-button', {
        onClick: () => handleEdit(row.id),
        disabled: !hasPermission(['/article/edit']),
      }, '编辑'),
    ])
  }
}
```

主模块权限标识与后端 API 路由路径一致（如 `/article/edit` 对应 `api/admin/article/article.go` 中 `g.Meta` 的 `path:"/article/edit"`）。

### 插件模块权限（⚠️ 需要拼接插件名称前缀）

插件的 API 路由前有插件名称作为前缀（如 `/test_plugin/article/edit`），`hasPermission` 中**必须带上这个前缀**：

```ts
// ❌ 错误：插件中不能直接用主模块的路径格式
hasPermission(['/article/edit'])

// ✅ 正确：必须拼接插件名称前缀
hasPermission(['/test_plugin/article/edit'])
```

**完整的插件列表页 model.ts 示例**：

```ts
// web/src/views/addons/test_plugin/article/model.ts
import { usePermission } from '@/hooks/web/usePermission'
const { hasPermission } = usePermission()

export const columns = [
  { title: 'ID', key: 'id', width: 80 },
  { title: '标题', key: 'title', width: 200 },
  {
    title: '操作',
    key: 'action',
    width: 250,
    render(row) {
      return h('div', [
        h('n-button', {
          onClick: () => handleEdit(row.id),
          // ⚠️ 插件格式：/插件名/接口路径
          disabled: !hasPermission(['/test_plugin/article/edit']),
        }, '编辑'),
        h('n-button', {
          onClick: () => handleDelete(row.id),
          disabled: !hasPermission(['/test_plugin/article/delete']),
        }, '删除'),
      ])
    }
  }
]
```

**插件 API 模块中的 URL 也需要拼接前缀**：

```ts
// web/src/api/addons/test_plugin/index.ts
import { http, jumpExport } from '@/utils/http/axios'

// ⚠️ 插件接口 URL = /插件名/接口路径
export function List(params) {
  return http.request({ url: '/test_plugin/article/list', method: 'get', params })
}
export function Edit(params) {
  return http.request({ url: '/test_plugin/article/edit', method: 'POST', params })
}
export function Delete(params) {
  return http.request({ url: '/test_plugin/article/delete', method: 'POST', params })
}
```

**权限标识与后端、菜单 SQL 三处对应关系**：

```
┌──────────────────────────────────────────────────────────┐
│ 1. 后端 API 定义（Go）                                    │
│    server/addons/test_plugin/api/admin/article/article.go│
│    g.Meta `path:"/test_plugin/article/edit" method:"post"`│
├──────────────────────────────────────────────────────────┤
│ 2. 菜单按钮 SQL（权限绑定）                                │
│    INSERT INTO hg_admin_menu (..., type, permissions)    │
│    VALUES (..., 3, '/test_plugin/article/edit')          │
├──────────────────────────────────────────────────────────┤
│ 3. 前端权限判断（Vue）                                     │
│    hasPermission(['/test_plugin/article/edit'])           │
│    http.request({ url: '/test_plugin/article/edit' })     │
└──────────────────────────────────────────────────────────┘
```

**三处必须完全一致**，都以 `/插件名/接口路径` 的格式书写。

## 4.12 路由与菜单

### 路由实例

```ts
// router/index.ts
import { createRouter, createWebHashHistory } from 'vue-router'
// ⭐ 使用 Hash 模式（#/xxx）
const router = createRouter({
  history: createWebHashHistory(),
  routes: [...]

```

### 菜单数据来自后端

**路由和菜单由后端菜单数据驱动**，不是在前端硬编码。用户登录后，后端返回该用户可见的菜单树，前端动态注册路由。

### 路由守卫

```ts
// router/guard.ts
router.beforeEach(async (to, from, next) => {
  const userStore = useUserStore()
  if (!userStore.token && to.path !== '/login') {
    next('/login')
    return
  }
  // 检查权限... 从后端获取动态路由...
  next()
})
```

## 4.14 状态管理（Pinia Store）

### 字典 Store

`store/modules/dict.ts`：

```ts
import { defineStore } from 'pinia'
import { getDictData } from '@/api/dict'

export const useDictStore = defineStore('dict', {
  state: () => ({
    dictMap: {} as Record<string, any[]>
  }),
  actions: {
    // ⭐ 加载字典数据
    async loadOptions(keys: string[]) {
      const res = await getDictData({ keys })
      res.forEach(item => { this.dictMap[item.key] = item.options })
    },
    // ⭐ 获取标签文本
    getLabel(key: string, value: any): string {
      const option = this.dictMap[key]?.find(o => o.value === value)
      return option?.label || String(value)
    },
    // ⭐ 获取选项列表
    getOptions(key: string): any[] {
      return this.dictMap[key] || []
    }
  }
})
```

## 4.15 国际化（Vue I18n）

```yaml
system:
```

```vue
<template>
  <!-- 基础翻译 -->
  <div>{{ t('你好，美丽世界') }}</div>
  <!-- → Hello, Beautiful World -->

  <!-- 格式化翻译 -->
  <div>{{ t('剩余{num}余额', {num: 100}) }}</div>

import { useI18n } from 'vue-i18n'
```
