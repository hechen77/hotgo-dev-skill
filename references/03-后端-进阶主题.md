## 3.1 插件系统（Addon）

### 插件定位
适合独立、临时性、工具类型的功能以插件形式开发（如小游戏、广告管理、文章管理、小程序、微商城等）。插件之间完全隔离，方便多项目复用。

### 插件目录结构

```
server/addons/<addonName>/
├── main.go           # 插件入口：定义 module 结构体，实现 Module 接口
├── global/
│   ├── global.go     # 插件全局变量
│   └── init.go       # Init(ctx, skeleton)，GetSkeleton()
├── api/              # API 契约（admin/api/home 子目录）
├── controller/       # 薄控制器（admin/api/home 子目录）
├── logic/
│   ├── logic.go      # 匿名导入各 logic 子包触发 init()
│   └── sys/          # 业务实现
├── service/          # 插件内接口声明（独立，不与主模块混用）
├── model/input/      # 插件内入参/出参
├── router/
│   ├── admin.go      # 后台路由
│   ├── api.go        # 前台 API 路由
│   ├── home.go       # 前台页面路由
│   └── genrouter/    # 模块路由追加
├── consts/           # 插件常量
├── crons/            # 定时任务（可选）
├── queues/           # 消息队列消费者（可选）
└── resource/         # 静态资源（可选）
```

### 插件入口 `main.go`

```go
package <addonName>

import (
    "context"
    "sync"
    "github.com/gogf/gf/v2/net/ghttp"
    _ "hotgo/addons/<addonName>/crons"
    "hotgo/addons/<addonName>/global"
    _ "hotgo/addons/<addonName>/logic"
    _ "hotgo/addons/<addonName>/queues"
    "hotgo/addons/<addonName>/router"
    "hotgo/internal/library/addons"
    "hotgo/internal/service"
)

type module struct {
    skeleton *addons.Skeleton
    ctx      context.Context
    sync.Mutex
}

func init() { newModule() }

func newModule() {
    m := &module{
        skeleton: &addons.Skeleton{
            Label:       "插件显示名",
            Name:        "<addonName>",  // 与目录名一致
            Group:       1,
            Brief:       "简介",
            Description: "详细描述",
            Author:      "作者",
            Version:     "v1.0.0",
        },
        ctx: gctx.New(),
    }
    addons.RegisterModule(m)
}

func (m *module) Start(option *addons.Option) (err error) {
    global.Init(m.ctx, m.skeleton)
    option.Server.Group("/", func(group *ghttp.RouterGroup) {
        group.Middleware(service.Middleware().Addon)
        router.Admin(m.ctx, group)
        router.Api(m.ctx, group)
    })
    return
}

// 以下方法可选实现（默认留空）
func (m *module) Stop() {}
func (m *module) Ctx() context.Context { return m.ctx }
func (m *module) GetSkeleton() *addons.Skeleton { return m.skeleton }
func (m *module) Install(ctx context.Context) error { return nil }
func (m *module) Upgrade(ctx context.Context) error { return nil }
func (m *module) UnInstall(ctx context.Context) error { return nil }
```


## 3.2 字典与枚举规范

**字典由后端统一维护**，前端通过固定接口拉取，禁止前端硬编码枚举值。

### 后端定义与注册

```go
// internal/consts/<module>.go

// 1. 定义常量
const (
    XxxStatusEnabled  = 1 // 启用
    XxxStatusDisabled = 2 // 禁用
)

// 2. 定义选项
var XxxStatusOptions = []*model.Option{
    dict.GenSuccessOption(XxxStatusEnabled, "启用"),
    dict.GenWarningOption(XxxStatusDisabled, "禁用"),
}

// 3. init() 注册
func init() {
    dict.RegisterEnums("XxxStatusOptions", "XXX状态选项", XxxStatusOptions)
}
```

### 样式函数选择

| 函数 | 适用场景 |
|------|---------|
| `dict.GenSuccessOption` | 正常、启用、成功 |
| `dict.GenWarningOption` | 待处理、禁用、警告 |
| `dict.GenErrorOption` | 失败、封禁、危险 |
| `dict.GenInfoOption` | 信息类、中性状态 |
| `dict.GenPrimaryOption` | 主要操作、强调 |
| `dict.GenDefaultOption` | 默认、无特殊含义 |
| `dict.GenHashOption` | 不确定数量的动态枚举 |

### 后端使用

```go
// 获取标签文本
label := dict.GetOptionLabel(consts.XxxStatusOptions, in.Status)

// 校验是否为合法值
if !dict.HasOptionKey(consts.XxxStatusOptions, in.Status) {
    return gerror.New("状态值不合法")
}
```

## 3.3 错误处理规范

### 创建错误

```go
import "github.com/gogf/gf/v2/errors/gerror"

// 普通业务错误（最常用）
return gerror.New("用户名已存在")

// 包装底层错误，追加上下文
if err := dao.User.Ctx(ctx).Data(data).Insert(); err != nil {
    return gerror.Wrap(err, "创建用户失败")
}

// 带格式化参数
return gerror.Newf("订单 %d 状态不合法", in.Id)
```

### 错误传递原则

- **logic 层**：遇到底层错误用 `gerror.Wrap` 追加上下文；纯业务校验用 `gerror.New`
- **controller 层**：直接 `return` 透传，**不包装、不打印日志**
- **⚠️ 关键**：命名返回值模式下设置 `err` 后必须 `return`，否则后续代码覆盖 err 或 panic

```go
// ❌ 错误
func (s *sXxx) Query(ctx context.Context, in *sysin.QueryInp) (res *sysin.Model, err error) {
    err = checkPermission(ctx)  // 设了 err
    // 继续执行 → 后续操作覆盖了 err 或触发 panic
    return dao.Xxx.Ctx(ctx).One()
}

// ✅ 正确
func (s *sXxx) Query(ctx context.Context, in *sysin.QueryInp) (res *sysin.Model, err error) {
    if err = checkPermission(ctx); err != nil {
        return  // 设了 err 后立即 return
    }
    return dao.Xxx.Ctx(ctx).One()
}
```

### 响应格式

统一由框架中间件处理，**无需手动包装**：

```json
{
    "code": 0,
    "message": "操作成功",
    "timestamp": 1684145107,
    "traceID": "084022730d495f17f19e550140f3e1a8",
    "data": {}
}
```

- 成功 `code: 0`，失败 `code: -1`（GoFrame 内置）
- `gerror.Wrap(err, "用户友好提示")` → 客户端看到最外层消息，开发者日志可见完整堆栈
- 禁止在 controller 层手动 `Write`/`WriteJSON` 破坏统一格式

## 3.4 缓存使用规范

**统一通过 `cache.Instance()` 操作**，禁止直接调 `g.Redis().Do()`。底层适配器由配置决定，业务层无需关心。

```go
import (
    "hotgo/internal/library/cache"
    "time"
)

// 设置
err := cache.Instance().Set(ctx, "user:info:1", userInfo, 30*time.Minute)

// 获取
val, err := cache.Instance().Get(ctx, "user:info:1")
if err != nil || val.IsNil() { /* 缓存未命中 */ }

// 删除
_, err = cache.Instance().Remove(ctx, "user:info:1")

// GetOrSet（推荐用于只读类缓存）
val, err := cache.Instance().GetOrSet(ctx, "user:info:1", func(ctx context.Context) (interface{}, error) {
    return dao.AdminMember.Ctx(ctx).Where(...).One()
}, 30*time.Minute)
```

### Cache Key 命名规范

- 格式：`模块:实体:唯一标识`，如 `user:info:1`、`config:site:basic`、`dict:options:XxxStatus`
- 同模块 Key 统一在 `internal/consts/cache.go` 中定义为常量
- 插件缓存 Key 加插件名前缀：`hgexample:order:1`
- **更新/删除数据后主动清除相关缓存**，不依赖过期
- 频繁变动的数据用短时间（5~30分钟），基本不变的数据可更长

## 3.5 日志规范

### 日志级别

| 级别 | 方法 | 适用场景 |
|------|------|---------|
| Info | `g.Log().Info/Infof` | 正常业务流程关键节点 |
| Warning | `g.Log().Warning/Warningf` | 不影响主流程的错误（重试、降级、外部服务异常） |
| Panic | `g.Log().Panic/Panicf` | 极其严重、必须立即终止的情况 |

### 使用示例

```go
// Info：正常流程
g.Log().Infof(ctx, "用户登录成功 uid:%d ip:%s", member.Id, ip)
g.Log().Info(ctx, "支付回调完成", g.Map{"orderId": orderId, "status": status})

// Warning：非致命错误但需要关注
g.Log().Warningf(ctx, "短信发送失败，已跳过 mobile:%s err:%+v", mobile, err)

// Panic：启动时核心功能不可用
g.Log().Panicf(ctx, "缓存未初始化，无法启动: %+v", err)
```

### 日志原则

- **必须传入 `ctx`**（第一个参数），便于链路追踪
- 关键日志携带足够的业务字段，用 `g.Map{}` 附加结构化信息
- **禁止** `fmt.Println` 替代日志
- **禁止**在 controller 层重复打印已被框架记录的 error

## 3.8 定时任务（Crons）

```go
// server/internal/crons/<name>.go（或 addons/<addonName>/crons/<name>.go）
package crons

import (
    "context"
    "hotgo/internal/library/cron"
)

func init() {
    cron.Register(MyTask)
}

var MyTask = &cMyTask{name: "myTask"} // name 与后台配置的任务名称一致

type cMyTask struct{ name string }

func (c *cMyTask) GetName() string { return c.name }

func (c *cMyTask) Execute(ctx context.Context, parser *cron.Parser) (err error) {
    parser.Logger.Infof(ctx, "任务执行：%v", time.Now())
    return
}
```

## 3.9 消息队列（Queues）

```go
// internal/consts/queue.go
const (
```

### 消费者

```go
// server/internal/queues/<name>.go（或 addons/<addonName>/queues/<name>.go）
package queues

import (
    "context"
    "encoding/json"
    "hotgo/internal/consts"
    "hotgo/internal/library/queue"
)

func init() {
    queue.RegisterConsumer(MyConsumer)
}

var MyConsumer = &qMyConsumer{}

type qMyConsumer struct{}

func (q *qMyConsumer) GetTopic() string { return consts.QueueMyTopic }

func (q *qMyConsumer) Handle(ctx context.Context, mqMsg queue.MqMsg) (err error) {
    var data MyData
    if err = json.Unmarshal(mqMsg.Body, &data); err != nil {
        return
    }
    // 处理消息...
    return
}
```

### 发送消息

```go
// 即时消息
queue.Push(consts.QueueMyTopic, data)

// 延迟消息（仅 redis/rocketmq 驱动支持）
// redis 第二个参数为延迟秒数
queue.SendDelayMsg(consts.QueueMyTopic, data, 10)
// rocketmq 第二个参数为延迟级别（1s/5s/10s/30s/...）
queue.SendDelayMsg(consts.QueueMyTopic, data, 2)
```

**支持驱动**：`disk`（默认）、`redis`、`rocketmq`、`kafka`，在 `config.yaml` 中配置。

## 3.14 实战案例：自定义 HTTP 请求日志（IHook 接口）

> **核心原则**：修改核心框架行为时，优先使用框架预留的扩展接口，而不是直接修改 `internal/` 下的源文件。

### 需求场景

HotGo 默认的访问日志格式无法满足需求——想要在每个 HTTP 请求结束后，输出一份结构清晰、包含"请求模块、操作用户、请求方法、接口路径、请求耗时、IP 地址、GET/POST 参数、报错信息、响应结果"的完整日志。

### ❌ 错误做法：直接改框架文件

```go
// ❌ 直接修改 internal/logic/hook/init.go 中的 AfterOutput 方法
// 框架升级时会被覆盖，导致需要手动合并代码
func (s *sHook) AfterOutput(r *ghttp.Request) {
    s.accessLog(r)
    s.lastAdminActive(r)
    logStr := utility.RequestLogBeautification(r) // 自己加的逻辑
    if logStr != "err" {
        g.Log().Infof(gctx.New(), "%s", logStr)
    }
}
```

### ✅ 正确做法：实现 IHook 接口并注册

**第一步：在 `utility/` 下写工具函数**

`server/utility/log_beautify.go`：

```go
package utility

import (
    "bytes"
    "encoding/json"
    "fmt"
    "reflect"
    "time"
    "github.com/gogf/gf/v2/net/ghttp"
    "github.com/gogf/gf/v2/util/gmeta"
    "hotgo/internal/library/contexts"
    "hotgo/internal/service"
)

// PrettyPrint JSON 格式化
func PrettyPrint(v interface{}) string {
    b, err := json.Marshal(v)
    if err != nil {
        return fmt.Sprint(v)
    }
    var out bytes.Buffer
    if err = json.Indent(&out, b, "", "  "); err != nil {
        return fmt.Sprint(v)
    }
    return out.String()
}

// RequestLogBeautification 请求日志结构化输出
func RequestLogBeautification(r *ghttp.Request) string {
    ctx := r.Context()
    ctxModel := contexts.Get(ctx)
    logData := service.SysLog().AnalysisLog(ctx)

    // 操作用户
    userName := "用户未登录"
    if user := contexts.GetUser(ctx); user != nil {
        userName = fmt.Sprintf("%s[%d]", user.Username, user.Id)
    }

    // 请求 handler 信息（tags + summary）
    handler := r.GetServeHandler().Handler
    if handler.Info.Type == nil || handler.Info.Type.NumIn() != 2 {
        return "err"
    }
    objectReq := reflect.New(handler.Info.Type.In(1))
    tags := gmeta.Get(objectReq, "tags")
    summary := gmeta.Get(objectReq, "summary")

    timestamp := time.Unix(logData.Timestamp, 0).Format("2006-01-02 15:04:05")

    return fmt.Sprintf(
        "\n——————————>  %s — %s  <——————————\n"+
            "请求模块：%s\n操作用户：%s\n请求方法：%s\n接口路径：%s\n"+
            "请求耗时：%s\nIP地址：%s\n响应时间：%s\n"+
            "GET数据：%s\nPOST数据：%s\n"+
            "报错信息：%s\n响应结果：%s\n"+
            "——————————请求完成——————————\n",
        tags, summary,
        logData.Module, userName, logData.Method, logData.Url,
        AutomaticTimeConversion(logData.TakeUpTime),
        logData.Ip, timestamp,
        PrettyPrint(logData.GetData), PrettyPrint(logData.PostData),
        logData.ErrorData.String(), PrettyPrint(ctx.Response),
    )
}

// AutomaticTimeConversion 毫秒 → 人类可读时间
func AutomaticTimeConversion(msStr string) string {
    d, err := time.ParseDuration(msStr + "ms")
    if err != nil {
        return "Invalid time"
    }
    if d > time.Minute {
        return fmt.Sprintf("%d分%d秒", int(d.Minutes()), int(d.Seconds())%60)
    }
    if d < time.Second {
        return fmt.Sprintf("%d毫秒", int(d.Milliseconds()))
    }
    return fmt.Sprintf("%d秒%d毫秒", int(d.Seconds()), int(d.Milliseconds())%1000)
}
```

## 3.15 实战案例：替换邮件发送驱动（ems.Drive 接口）

### 需求场景

HotGo 默认使用 Go 标准库 `net/smtp` 发送邮件，但测试中发现长时间无法连接 SMTP 服务器（报 EOF），且缺乏 SSL 加密、连接超时等现代邮件发送所需的能力。需要替换为更成熟稳定的第三方库（如 `go-simple-mail`）。

### ❌ 错误做法：直接改框架内部实现

```go
// ❌ 直接修改 internal/library/ems/ems.go 中的 sendToMail 函数
// 把 net/smtp 替换为 go-simple-mail
// 框架升级 → 改动丢失 → 需要手动合并
func sendToMail(config *model.EmailConfig, to, subject, body, mailType string) error {
    // 把原来的 smtp.SendMail() 删掉，换成 mail.NewSMTPClient()...
}
```

### ✅ 正确做法：实现 Drive 接口，注册新驱动

首先看 HotGo 已有的邮件驱动接口。在 `server/internal/library/ems/ems.go` 中搜索驱动抽象：

```go
// HotGo 邮件驱动接口（简化示意，实际路径见源码）
type Drive interface {
    Send(ctx context.Context, in *SendInp) error
}
```

**第一步：封装 go-simple-mail 驱动**

`server/internal/library/ems/simplemail_driver.go`：

```go
package ems

import (
    "context"
    "fmt"
    "strings"
    "time"

    "github.com/gogf/gf/v2/errors/gerror"
    "github.com/gogf/gf/v2/frame/g"
    "github.com/xhit/go-simple-mail/v2"
    "hotgo/internal/model"
    "hotgo/utility/validate"
)

// SimpleMailDriver go-simple-mail 邮件驱动
type SimpleMailDriver struct{}

func init() {
    // ⭐ 注册新驱动，不修改任何现有代码
    RegisterDrive("simplemail", NewSimpleMailDriver)
}

func NewSimpleMailDriver() Drive {
    return &SimpleMailDriver{}
}

func (d *SimpleMailDriver) Send(ctx context.Context, in *SendInp) error {
    config := in.Config
    if config == nil {
        return gerror.New("邮件配置不能为空")
    }

    // 创建 SMTP 客户端
    smtpServer := mail.NewSMTPClient()
    smtpServer.Host = config.Host
    smtpServer.Port = int(config.Port)
    smtpServer.Username = config.User
    smtpServer.Password = config.Password
    smtpServer.Encryption = mail.EncryptionSSL
    smtpServer.ConnectTimeout = 10 * time.Second
    smtpServer.SendTimeout = 10 * time.Second

    // 连接 SMTP
    smtpClient, err := smtpServer.Connect()
    if err != nil {
        g.Log().Errorf(ctx, "SMTP 连接失败: %v", err)
        return gerror.Wrap(err, "邮件服务器连接失败")
    }
    defer smtpClient.Close()

    // 批量发送
    sendTo := strings.Split(in.To, ";")
    for _, em := range sendTo {
        if !validate.IsEmail(em) {
            return gerror.Newf("邮件格式不正确：%v", em)
        }

        email := mail.NewMSG()
        email.SetFrom(fmt.Sprintf("%s <%s>", config.SendName, config.User)).
            AddTo(em).
            SetSubject(in.Subject)

        if in.MailType == "html" {
            email.SetBody(mail.TextHTML, in.Body)
        } else {
            email.SetBody(mail.TextPlain, in.Body)
        }

        if err = email.Send(smtpClient); err != nil {
            g.Log().Errorf(ctx, "邮件发送失败: %v", err)
            return err
        }
    }
    return nil
}
```

**第二步：在后台配置中切换驱动**

HotGo 后台 → 系统设置 → 配置管理 → 邮件设置 → 驱动选择 `simplemail`。

框架内部通过 `ems.New(config.Driver)` 根据驱动名查找对应的驱动实现，你新增的 `simplemail` 通过 `init()` 中的 `RegisterDrive()` 自动注册，完全不需要修改 `ems.go` 的原有代码。

### 驱动替换通用模式

HotGo 中短信、缓存、队列、存储等基础设施都遵循同样的接口+注册模式：

```go
// 通用模式
type XxxDrive interface {
    Do(ctx context.Context, in *XxxInp) error
}

// 注册新驱动（在 init 中调用）
RegisterXxxDrive("my-driver", func() XxxDrive { return &MyDriver{} })

// 配置切换（在 config.yaml 或后台配置中）
xxx:
  driver: "my-driver"   // 从 default → my-driver
```

**一个 `init()` + 一个接口实现**，不碰任何核心代码，框架升级完全不受影响。
